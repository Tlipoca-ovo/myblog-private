# 图片使用规范

本项目部署在 Cloudflare Workers/D1 环境中，**图片不存储在本地**，而是上传到图床后，将图床返回的 URL 链接填入博客系统。因此在使用前需要先准备好符合尺寸要求的图片。

---

## 图片规格一览

| 使用场景 | 最大宽度 | 建议尺寸 | 宽高比 | 推荐格式 | 单张大小上限 |
|----------|----------|----------|--------|----------|--------------|
| **文章封面图** | 1200px | 1200×675 | 16:9 | WebP / JPG | 500KB |
| **文章内容图片** | 900px | 900×auto | 自适应 | WebP / PNG / JPG | 300KB |
| **友链 Logo** | 80px | 40×40 / 80×80 | 1:1（正方形） | PNG / WebP | 50KB |
| **作者头像** | 200px | 100×100 / 200×200 | 1:1（正方形） | PNG / WebP | 30KB |

---

## 详细说明

### 文章封面图

用于文章列表页（卡片缩略图）和文章详情页顶部。

- **最大宽度**：1200px
- **建议尺寸**：1200×675px（16:9 比例）
- **上传前处理**：
  1. 将图片宽度调整为 **1200px**
  2. 按 16:9 比例裁切高度（约 675px）
  3. 压缩至 **500KB 以内**
- **图床工具推荐**：TinyPNG（压缩）、PicGo（上传）

### 文章内容图片

用于 Tiptap 编辑器中插入到文章正文内的图片。

- **最大宽度**：900px
- **建议尺寸**：宽度 900px，高度自适应
- **上传前处理**：
  1. 将图片宽度调整为 **900px**（编辑器会自动限制最大宽度）
  2. 压缩至 **300KB 以内**
- **建议**：文章内图片不宜过多，每篇建议不超过 5 张，减少加载时间

### 友链 Logo

友链页面中显示的网站图标。

- **建议尺寸**：40×40px 或 80×80px（正方形）
- **宽高比**：1:1
- **上传前处理**：
  1. 裁切为正方形
  2. 建议使用透明背景 PNG
  3. 压缩至 **50KB 以内**

### 作者头像

管理后台账号设置中的头像显示。

- **建议尺寸**：100×100px 或 200×200px（正方形）
- **宽高比**：1:1
- **上传前处理**：裁切为正方形，压缩至 **30KB 以内**

## 图床推荐

| 图床 | 免费额度 | 说明 |
|------|----------|------|
| **ImgBB** | 无限（单张 ≤ 32MB） | 无需注册，即可用，支持批量上传 |
| **SM.MS** | 无限（单张 ≤ 5MB） | 无需注册，API 上传 |
| **PicGo + 图床插件** | 取决于图床 | 本地工具，支持拖拽上传到多种图床 |
| **Cloudflare R2** | 付费方案 | 自托管，无第三方依赖 |

> **推荐流程**：本地使用 [PicGo](https://picgo.github.io/PicGo-Doc/) 配合 ImgBB 或 SM.MS，配置好后直接拖拽上传，复制返回的 URL 粘贴到博客系统。

## 压缩工具推荐

| 工具 | 平台 | 说明 |
|------|------|------|
| **TinyPNG** | 在线 | https://tinypng.com，压缩率高，无需安装 |
| **Squoosh** | 在线 / 本地 | Google 出品，支持 WebP 转换 |
| **PicGo** | 本地 | 支持上传时自动压缩（需安装插件） |
| **Sharp** | 命令行 | Node.js 图片处理，适合批量处理 |

## 快速处理命令（可选）

如果你使用 Node.js，可以用 `sharp` 批量处理当前目录下的图片：

```bash
# 安装
npm install -g sharp

# 将 src/images 目录下的所有图片压缩并调整为最大宽度 1200px
node -e "
const sharp = require('sharp');
const fs = require('fs');
const path = require('path');
const dir = './src/images';
fs.readdirSync(dir).forEach(f => {
  sharp(path.join(dir, f))
    .resize(1200, null, { withoutEnlargement: true })
    .jpeg({ quality: 80 })
    .toFile(path.join(dir, 'optimized', f));
});
"
```
